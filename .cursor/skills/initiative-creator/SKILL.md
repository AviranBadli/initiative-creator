# Initiative Creator — Skill System

Help engineers and managers in the Model Validation & Benchmarking team create, review, and submit high-quality JIRA Initiatives from a problem statement to a live ticket, with full pipeline support.

---

## When to Use This Skill

Use the initiative-creator system when you want to:

- Turn a rough idea or problem statement into a complete JIRA Initiative
- Review and improve an existing initiative draft or a live JIRA ticket
- Submit a reviewed initiative directly to JIRA via MCP
- Run the full pipeline end-to-end with minimal friction
- Break an approved initiative down into structured Epics

**Trigger phrases:**
- "Create a new initiative for..."
- "I want to write up an initiative about..."
- "Let's do a full initiative for..."
- "Review this initiative draft"
- "Submit initiative to JIRA"
- "Break this initiative into epics"
- `/initiative.create`, `/initiative.review`, `/initiative.submit`, `/initiative.speedrun`, `/initiative.breakdown`

---

## Pipeline Overview

```
Problem Statement
      ↓
/initiative.create   →  artifacts/initiative-{slug}-{date}.md
      ↓
/initiative.review   →  scored + revised draft (up to 2 auto-revise cycles)
      ↓
/initiative.submit   →  JIRA Initiative (JN-XXXX) via Atlassian MCP
      ↓
/initiative.breakdown → Epic drafts (3-7 epics saved as artifact files)

/initiative.speedrun →  runs create → review → submit with minimal interaction
```

## Artifacts Convention

All initiative drafts are saved to:
```
artifacts/initiatives/initiative-{slug}-{date}.md
```

Epic drafts generated by `breakdown` are saved to:
```
artifacts/initiatives/epics/epic-{slug}-{n}.md
```

### Frontmatter Schema

Every artifact file uses this frontmatter:
```yaml
---
status: draft | reviewed | submitted
jira_key: ""          # filled in by submit skill after successful creation
created: YYYY-MM-DD
title: ""             # mirrors the JIRA Summary field
assignee: ""          # JIRA account ID
priority: Medium      # Critical | High | Medium | Low | Nice to Have
---
```

## Quality Bar

Before an initiative leaves `draft` status, it MUST have:

- [ ] Title starts with an action verb and is 60-80 characters
- [ ] Context explains WHY (not just WHAT) in 2-4 paragraphs
- [ ] 3-6 specific, measurable goals
- [ ] Scope section with explicit in-scope AND out-of-scope items
- [ ] Definition of Done with verifiable, non-ambiguous criteria
- [ ] Q&A section (even if only Open Questions with no answers yet)
- [ ] Current date on all timestamps (from `mcp_time_get_current_time`)

If any item is missing → `review` skill flags and auto-revises before `submit` is allowed.
